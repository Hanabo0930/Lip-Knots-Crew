# CHANGELOG v3.1

- 本番移行リハーサルを7工程で管理するサーバー機能を追加
- Firestore、Storage、Authのバックアップ完了証跡を必須化
- stagingと同一Projectへの復元を拒否し、隔離Projectだけを許可
- 元・復元の件数、snapshot SHA-256、標本差異、権限probe、smokeを検算
- 移行dry-runの予定件数、適用件数、差異0件を必須化
- 切戻しRTO 60分以内、RPO 5分以内、切戻し後smoke 0件を必須化
- リハーサル証跡7件以上とSHA-256 fingerprintを固定
- 合格証跡を本番公開ゲートのバックアップ・復元・移行・切戻しへ自動連携
- 手動チェックだけでは上記4条件を通過できないようサーバー側で上書き
- 安全な9工程CLI previewと12ケースの隔離設定拒否テストを追加
- 復元・移行・切戻し判定コア20ケースを追加
