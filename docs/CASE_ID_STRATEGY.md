# 案件ID設計 v0.2

## 現段階

シフト表には案件ID列がないため、Firestore内部だけで案件IDを生成します。

生成材料:

- 会社ID
- スプレッドシートID
- 月別タブ名
- 実施日
- クライアント名
- 店舗名
- 実施時間
- 同一内容の行が複数ある場合の出現順

スタッフ名、単価、手当、メーカー、メニューはID材料に含めません。
そのため、手配・金額・商品変更ではIDが変わりにくい構造です。

## 制約

完全に同じ日付・クライアント・店舗・時間の行を途中へ挿入すると、同一枠内の出現順が変わる可能性があります。

## 最終形

既存GASの安全監査後、右端へ非表示の案件ID列を追加します。
一度スプシへ案件IDを保存した後は、そのIDを最優先で使用するため、並べ替えや行追加でも案件の紐付けが維持されます。
