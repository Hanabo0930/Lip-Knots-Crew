# 端末管理 v0.4

## スタッフ側

- 初回ログイン後、端末IDをブラウザ内に作成
- 端末名、OS、ブラウザ情報、最終利用日時を保存
- マイページで端末一覧を確認
- 使っていない端末を個別ログアウト
- 全端末ログアウトも可能

## 管理者側

- スタッフごとの端末一覧
- 全端末の強制ログアウト
- 個別端末の停止
- 監査ログ

## 失効方法

個別端末:
- `deviceSessions.active=false`
- 通常アプリはリアルタイム監視し、その端末で自動サインアウト

全端末:
- 端末セッションを全停止
- Firebase Admin SDKで更新トークンを失効
- 次回トークン更新時までに全端末をログアウト

## セキュリティ上の補足

ブラウザ単位の完全な即時失効をさらに強くする場合は、将来、端末別カスタムセッション方式へ移行します。
Phase 1では通常のアプリ利用に対して十分な端末管理とし、全端末停止はFirebaseのハード失効を使用します。
