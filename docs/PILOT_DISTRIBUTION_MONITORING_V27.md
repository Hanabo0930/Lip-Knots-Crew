# パイロット配布・自動監視 v2.7

## 前提

- v2.6手順で実stagingが`GO`
- Go判定済みcheckpointのRelease IDを確認済み
- Firestore indexes・Functions・管理画面をv2.7でstagingへデプロイ済み
- Gmail送信SecretとログインURL設定済み
- 管理者端末のPush通知がON
- 本番導入チェックが合格、未解決キュー0件

## 開始

1. 管理画面「未ログイン者への案内」で対象者を抽出
2. 有効・メール登録済みスタッフを3～5名だけ選択
3. Go判定済みRelease IDと期間1～14日を入力
4. 「パイロット開始」を押して確認

開始時にサーバーが参加者・環境・安全設定・未解決キューを再検査します。合格後だけワンタイムログイン案内を配布し、5分監視を開始します。1名でも配布できない場合は`blocked`で停止します。

## 自動監視10項目

- スプシ書込block・dead letter
- 行追加block・dead letter
- 行追加・月次作成の手動確認キュー
- パイロット期間中の通知配信error
- パイロット招待失敗
- 再試行中キュー
- 24時間経過後の未ログイン参加者
- 利用停止参加者

判定は`CONTINUE`・`WATCH`・`PAUSE`です。criticalが1件でもあればPAUSE、warningだけならWATCHです。異常、状態変化、復旧、監視処理失敗を管理者Pushと監査証跡へ保存します。同一状態の再通知は既定30分cooldownです。

## 終了

期間終了時は自動で`review_required`になります。次段階へは自動拡大しません。v2.8では`PILOT_RESULT_EXPANSION_GATE_V28.md`に従い結果を提出し、提出者とは別の管理者による承認後だけ30～50名段階を判断します。

緊急時は管理画面の「停止」を使用します。参加者アカウント無効化や本番デプロイは自動実行しません。
