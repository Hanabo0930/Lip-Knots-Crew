# パイロット結果・30～50名移行承認ゲート v2.8

## 目的

3～5名パイロットの完了後、監視証跡と業務結果を固定条件で判定し、結果提出者とは別の管理者が承認するまで30～50名への移行をロックします。承認しても配布は自動実行されません。

## 自動集計

- パイロット完了状態、参加者3～5名、実施7日以上
- 招待失敗、5分監視run数、期待run数、coverage 90%以上
- 監視処理失敗、PAUSE、WATCH、最終監視状態

## 管理者入力

- 検証案件10件以上、完了率95%以上
- 請求・給与差額0円
- ダブルブッキング、メール対象差異、PDF差異、手動確認キュー各0件
- サポート問合せ5件以下（超過はwarning）
- 証拠参照1件以上、備考

## 判定と承認

1. 期間終了後、管理画面の「30～50名 移行承認ゲート」で結果を入力
2. 「結果を確定して判定」で自動集計と19項目を判定
3. blockerがあれば`expansion_blocked`。修正後に再提出
4. 合格時は`expansion_review_pending`。提出者本人は承認不可
5. 別の管理者が証拠を確認し、理由を入力して承認または否認
6. 承認時に自動値を再集計し、提出時のSHA-256 fingerprintと一致した場合だけ`expansion_approved`

承認・否認、提出者・承認者、判定fingerprint、理由はFirestoreの専用証跡と`auditLogs`へ保存します。

## 安全境界

- `expansion_approved`は30～50名段階への移行許可証跡であり、配布命令ではない
- 管理画面・Functionsとも30～50名への一括配布は実装しない
- v2.9の`STAGED_ROLLOUT_CONTROL_V29.md`に従い、承認証跡を一度だけ使用して3wave段階配布を作成する
- 監視処理失敗またはPAUSEが1件でもあれば自動block
