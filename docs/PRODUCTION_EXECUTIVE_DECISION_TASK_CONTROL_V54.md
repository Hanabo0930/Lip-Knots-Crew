# v5.4 経営判断タスク・期限・完了統制

## 自動作成

毎週火曜09:00 JSTの経営レポート配信時、判断事項を最大8件まで`executiveReports/{weekKey}/decisionTasks/{taskId}`へ作成します。同一週・同一判断は決定的IDで重複作成しません。

- CRITICAL: 24時間
- WATCH／高優先: 72時間
- 通常: 7日
- 担当: `ADMIN_EMAILS`内で決定的に負荷分散

## 操作権限

- 開始・完了: 現担当本人のみ
- 再開・再割当・期限変更: 本番管理者
- 再割当先: 同一企業の管理者allowlistのみ
- 期限: 現在より後、最大180日
- 理由・進捗・結果: 10～500文字

クライアントからFirestoreへ直接書き込めません。全操作をCallable経由でイベントと監査ログへ固定します。

## 自動督促

毎時監視し、完了済みを除いて同一日・同一段階の重複通知を防ぎます。

- 期限24時間以内: 期限接近通知
- 期限超過: LEVEL 1
- 24時間超過: LEVEL 2、管理者へ昇格
- 72時間超過: LEVEL 3、緊急通知

## 管理画面

`EXECUTIVE DECISION CONTROL`に直近8週の完了率、期限内完了率、進行中、期限超過、LEVEL 3、担当、期限、進捗・完了証跡を表示します。タスクPDF／CSVはUIDを含まず、CSV式注入を無害化します。

## 本番確認

1. `ADMIN_EMAILS`とFirebase Authの管理者UIDを確認
2. 火曜配信後に判断事項がタスク化されたことを確認
3. 担当本人で開始・完了を記録
4. テスト用期限でLEVEL 1～3通知と管理者昇格を確認
5. タスクPDF／CSVと監査ログを保管
