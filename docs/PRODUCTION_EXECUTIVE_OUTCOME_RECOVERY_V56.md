# v5.6 成果未達・改善再実行統制

## 判定

- unmeasured: 測定設計を再構築
- negative: 実行手順を即時是正
- below_target + 赤字ROI: 採算・投資を再設計
- below_target: 工程差分を分解して再実行
- effective / excellent: 改善不要、標準化を推奨

## 改善再実行

1. サーバー側で成果をrecovery_v1診断
2. 管理者が主原因・根拠・改善案・期限を固定
3. 元タスクの担当者を継承
4. 決定的IDの再実行タスクを作成
5. 担当者通知、イベント、監査ログを保存
6. 通常の開始・完了・期限督促・成果測定へ接続

同じ計画の再送は指紋で重複処理されません。
