# v5.3 経営レポート既読・督促・履歴運用

## 自動運用

- 毎週火曜09:00 JST以降に`ADMIN_EMAILS`へ経営レポートを配信します。
- 同一週は`deliveryKey`で重複配信しません。
- 本人未読は24・48・72時間でLEVEL 1→3へ昇格し、同日重複通知を拒否します。
- LEVEL 3は本人通知に加えて全管理者へ緊急通知します。
- Firebase Auth未解決の配信先は毎時再照合し、解決後に本人へ再通知します。

## 管理画面

- 「自分の経営レポートを既読」はログイン本人の配信分だけ更新します。
- 「代理確認」は配信先未解決またはLEVEL 3未読だけ実行でき、10文字以上の理由が必須です。
- 直近8週の配信・既読・代理・未読・未解決・LEVEL 3を表示します。
- 「履歴PDF」は最新週の配信先状態を含むA4横の印刷画面を開きます。

## 安全性

- Callableはproduction、管理者、企業一致、配信先一致を再検証します。
- Firestoreクライアント直接書込は拒否します。
- 既読、代理、復旧、督促、昇格はイベントと監査ログへ保存します。
- 管理画面レスポンスとPDFにはUIDを含めません。
