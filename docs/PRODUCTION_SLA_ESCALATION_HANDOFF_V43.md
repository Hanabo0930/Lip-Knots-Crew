# 本番SLA・自動エスカレーション・担当引継ぎ v4.3

## SLA

- CRITICAL: 確認5分、対応開始10分、引継ぎ受領10分
- WATCH: 確認15分、対応開始30分、引継ぎ受領30分
- 監視間隔: 1分

## 自動エスカレーション

- LEVEL 1: 確認期限接近
- LEVEL 2: 確認期限到達、対応開始待ち、引継ぎ受領期限接近
- LEVEL 3: 対応開始または引継ぎ受領SLA超過

同一`stage:level`は一度だけ通知し、段階が変わった場合だけ新しいeventとPushを生成します。CRITICALまたはLEVEL 3は静穏時間をバイパスします。

## 担当引継ぎ

1. 現担当者が`担当を引継ぐ`を選択。
2. `ADMIN_EMAILS`に登録済みの引継ぎ先メールと5文字以上のメモを入力。
3. サーバーが最新fingerprint、現在担当者、allowlist、自己引継ぎを再検証。
4. 新担当者が`引継ぎを受領`。
5. 前担当、新担当、メモ、時刻、回数、受領をevent・auditへ固定。

第三者の受領、未登録管理者、古い・解決済みアラート、現担当者以外の引継ぎは拒否されます。
