# 本番SLA実績・週次レビュー運用 v4.4

## 集計

- 対象期間: JST基準の当日を含む直近7日
- 確認SLA: CRITICAL 5分 / WATCH 15分
- 対応開始SLA: CRITICAL 10分 / WATCH 30分
- 表示: 達成率、平均確認・対応・解決、日別推移、原因、担当負荷
- 評価: 95点以上A、85点以上B、70点以上C、それ未満D

## 週次運用

1. 管理画面で総合評価、超過、対応中、レビュー未完了を確認する。
2. 原因ホットスポットと担当負荷から優先順位を決める。
3. `週次レポートをコピー`で定例報告へ転記する。
4. SLA超過またはLEVEL 3のアラートに再発防止レビューを登録する。
5. 原因・対策・責任者・期限を確認し、実施後に完了へ更新する。

## 安全条件

- production環境・管理者・同一企業のみ操作可能。
- 責任者は`ADMIN_EMAILS` allowlist登録済み管理者のみ。
- 未完了レビューの期限は現在より後、180日以内。
- Firestoreクライアント直接書込は禁止。
- 保存・完了はイベントと監査ログへ記録する。
- 実デプロイ、実通知、rollbackは画面から自動実行しない。
