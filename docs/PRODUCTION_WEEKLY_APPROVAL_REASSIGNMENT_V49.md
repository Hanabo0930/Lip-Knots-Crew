# 週次責任者の負荷分散・副担当・承認期限自動再割当

## 自動運用

- 未解決または既読SLA残り4時間の対象を毎時検出します。
- 未完了の担当件数が少ない管理者を主担当、次点を副担当にします。
- 同じ負荷の場合は対象・週・候補メールから決定的に順位を固定します。
- 対象本人と過去担当は候補から除外します。
- 初回割当時に主担当と副担当へ個別通知します。

## 承認期限

- 初回期限は既読SLAまでを基本に、1～4時間で設定します。
- 期限切れ時は副担当を優先し、使えなければ次の低負荷候補へ引き継ぎます。
- 再割当後の承認期限は2時間です。
- 自動再割当は最大2回です。
- 上限到達または候補枯渇時は管理者全体へ緊急移管します。

## 安全条件

- 現在の主担当本人だけが理由付き承認できます。
- 本人既読・代理確認・承認済みは再割当しません。
- 再割当直前にFirestore transactionで現在担当・状態・期限を再検証します。
- 緊急移管はescalation keyで同一段階を一度だけ通知します。
- 割当、再割当、承認、緊急移管をイベントと監査ログへ保存します。

## 管理画面

配信先ごとに主担当、副担当、両者の割当前負荷、承認期限、再割当回数、過去担当、承認フロー状態を表示します。
