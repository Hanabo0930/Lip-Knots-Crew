# 新規行の検算と自動ロールバック v1.1

## 書込後に確認するもの

- 案件IDが各行へ正しく入った
- AA列の数式
- AJ列の数式
- AR列の数式
- BB列の数式
- 設定した入力規則
- 条件付き書式

## 異常時

案件IDが想定どおり並んでいることを再確認してから、追加した行だけを削除します。

削除条件:

- 追加行の件数が一致
- 案件IDの順番が一致
- 他の人が追加行を編集していない

条件を満たさない場合は、勝手に削除しません。
`sheetRowManualInterventions` へ記録し、管理者の確認対象にします。

## 再試行

通信障害や一時的なGoogle APIエラーは最大5回再試行します。

次は自動再試行しません。

- 設定不足
- 月別タブなし
- 数式不足
- 入力規則不足
- 案件ID不一致
- 他の操作との競合
