# Google Sheets 読取専用同期 v0.2

## この版の目的

シフト表のセル・列・数式・GASを一切変更せず、Google Sheets APIの読取専用権限で月別タブを読み込み、Firestoreの `jobs` へ同期します。

## 実装済み

- `2026.7` のような月別タブを自動抽出
- 2025.10以降などの開始月指定
- J列相当が空白の区切り行を除外
- 募集中・手配済み・募集停止・キャンセル・下書きの判定
- M列の括弧内を応募条件として分離
- B列・J列の「（キャンセル）」表記を認識
- G列・H列の事前連絡を読取
- S～Zの最初の金額を基本単価候補として読取
- AA・AR・BBの概算値を読取
- Firestore内部案件IDの自動生成
- 既存スタッフ名との照合
- 手配済みスタッフの同日ロック作成
- 管理者によるプレビューと本同期
- 5分ごとの自動同期
- 重複実行防止ロック
- 同期履歴

## 安全な導入順

1. `sheetImportConfigs/lipknots` を `enabled:false` で登録
2. Cloud Functionsの実行用サービスアカウントへ対象スプシの閲覧権限を付与
3. 管理画面から「プレビュー」
4. 件数・タブ・行・スタッフ照合エラーを確認
5. `enabled:true` にする
6. 管理画面から「Firestoreへ同期」
7. スタッフ画面ではまだ応募を有効にせず、表示だけ検証
8. 問題がなければ `scheduleEnabled:true`

## スプレッドシートへの変更

この同期処理は、スプレッドシートへ書き込みません。

- 案件ID列を追加しない
- 名前を入力しない
- 色を変えない
- 数式を変更しない
- 行を移動しない

応募・事前連絡などの書込処理は、既存GAS監査と列マッピング確定後に別工程で有効化します。
