# スタッフ管理スプシ 読取専用同期 v0.3

## 対象タブ

- マスタ: 首都圏・北関東
- 東北: 東北
- 抹消: 完全に参照対象外

## 現行列

- B: 氏名
- G: 自宅都道府県
- P: 最寄り駅
- Q: 生年月日
- S: メールアドレス
- T: 電話番号

## 同じスタッフが2行ある場合

氏名を空白除去・全角半角統一して照合し、1つのスタッフアカウントへ統合します。

例:

- 山田花子 / first@example.com
- 山田花子 / second@example.com

同期後:

- スタッフアカウント: 1件
- ログイン可能メール: 2件
- シフト・提出・給与明細・履歴: 共通

## 安全性

この同期はスタッフ管理スプシへ書き込みません。

- 行を移動しない
- 抹消タブを読まない
- メールを変更しない
- 評価メモを変更しない
- 列を追加しない

## 利用停止

最初の同期では `markMissingInactive:false` にします。

全スタッフ件数・メール統合・エラーを確認した後で `true` にすると、マスタ・東北から消えた人を利用停止にできます。

利用停止時:

- 過去履歴は残す
- emailIndexを無効化
- Firestoreルールでアクセス拒否
- ログイン中端末の更新トークンを失効
- 復帰時は同じ氏名の内部IDで履歴を引き継ぐ
