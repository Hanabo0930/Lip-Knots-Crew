# 30～50名 3wave段階配布・停止制御 v2.9

## 前提

- v2.8の3～5名パイロット結果が合格
- 結果提出者とは別の管理者による`staff_30_50`承認が保存済み
- v2.9をstagingへデプロイ済み
- 管理者PushとGmail送信が実疎通済み

## 準備

1. 管理画面で有効・メール登録済みスタッフを30～50名選択
2. 観察時間24時間、連続CONTINUE 12runを確認
3. 「3waveへ固定」を実行

サーバーは二者承認、企業、参加者、同時配布を再検証します。承認証跡は一回だけ使用でき、準備だけではメールを送りません。

## wave計画

- 30名：10→10→10
- 40名：10→15→15
- 50名：10→20→20

wave 1は手動開始です。以後も自動配布せず、管理画面で明示的に開始します。

## 次waveの解放条件

- rollout状態が`ready`
- 有効な二者承認が継続
- 前waveの配布済み人数が計画と一致
- 24時間以上の観察
- 直近まで連続12run以上`CONTINUE`
- PAUSE、監視処理失敗、招待失敗が各0件
- 最終監視状態が`CONTINUE`

## 5分監視

- スプシ書込block・dead letter
- 行追加block・dead letter
- 手動確認キュー
- wave開始後の通知配信error
- 招待失敗
- 再試行中キュー
- 観察時間到達後の未ログイン参加者
- 利用停止参加者

WATCHは連続CONTINUEを0へ戻します。PAUSE、監視処理失敗、招待失敗は段階配布を永久ロックし、自動再開しません。

## 停止

管理画面の「即時停止」は`ready`・`observing`・配布処理中・`paused`で使用できます。配布処理中は現在処理中のスタッフ完了後、次スタッフへの送信前に中断します。送信済みメールは取消できません。途中成功・失敗・中止人数は監査証跡へ保存します。

停止後は同じrolloutを再開できません。再実施には新しい3～5名パイロット結果と二者承認が必要です。
