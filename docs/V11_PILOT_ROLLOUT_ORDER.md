# v1.1 パイロット導入順

## Phase 0 準備

1. 本番スプシのコピーを作成
2. 既存GASをコピー側で確認
3. 案件ID列を右端へ追加
4. 列マッピングを実物に合わせる
5. rowCreation.enabledはfalse

## Phase 1 事前検査

1. 管理画面の「本番導入チェック」
2. 対象月と追加予定人数を指定
3. 「行追加を事前検査」
4. 数式・入力規則・雛形行を確認
5. エラーをゼロにする

## Phase 2 検証コピー

1. 安全書込をON
2. rowCreation.enabledをON
3. adminJobCreationSourceReadyをON
4. 下書き案件を1件作成
5. 新規行と案件IDを確認
6. AA・AJ・AR・BBの数式を確認
7. 条件付き書式・入力規則を確認
8. 請求・給与・PDF・メールを旧版と突合

## Phase 3 少人数

1. テストスタッフ3～5名
2. 1週間
3. 10～20案件
4. 応募、事前連絡、提出、経費を確認
5. 競合・再試行・ロールバック履歴を確認

## Phase 4 拡大

1. 30～50名
2. 首都圏の一部案件
3. 東北は別タイミング
4. 問題がなければ全体へ拡大

- 書込エラー0件
- 手動確認0件
- 金額差異0円
- PDF差異なし
- メール送信対象差異なし
- ダブルブッキングなし
