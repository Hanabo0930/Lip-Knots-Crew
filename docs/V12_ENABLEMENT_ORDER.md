# v1.2 有効化順

## 1. 高設定のまま行う工程

- 導入ウィザード
- 読取専用検査
- 設定JSON作成
- 新月タブの事前検査
- 検証コピーでのテスト設計

## 2. 非常に高いへ切り替える工程

次へ進む前に、社長へ切替を案内します。

- 既存GASの全コード監査
- 本番スプシへ初めて書き込む直前
- 案件ID列や新規列を本番へ追加
- 請求・給与・PDF・メールの最終突合
- Firestore権限の本番最終監査

## 3. 検証コピーでの作業

1. スプシのコピーを作る
2. 導入ウィザードで検査
3. 設定下書きを保存
4. `verifiedSpreadsheetId` にコピー側IDを設定
5. 安全書込ON
6. 新月作成ON
7. 会社設定ON
8. 1か月分を作成
9. 数式・書式・保護・GASを確認
10. 異常時の自動削除を確認
